# Test Hints

## Happy Path

- [ ] Test basic functionality works as expected

## Edge Cases

- [ ] Empty input handling
- [ ] Invalid input handling
- [ ] Boundary values (max, min, zero)
- [ ] Null/nil handling

## Error Handling

- [ ] Graceful failure on bad input
- [ ] Proper error messages returned
- [ ] No panics or crashes

## Code-Specific

- [ ] Functions are testable in isolation
- [ ] Side effects are controlled/mockable
- [ ] Public API behaves correctly

## Security Considerations

- [ ] Input validation prevents injection
- [ ] No sensitive data exposed in logs/errors
- [ ] Authentication/authorization checks (if applicable)
