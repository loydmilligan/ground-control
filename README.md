# Ground Control

> Vibe coding project management — where AI agents orchestrate task flow, and projects go from idea to deployed.

[![Go Version](https://img.shields.io/badge/Go-1.25+-00ADD8?style=flat&logo=go)](https://go.dev/)
[![License](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

## What is Ground Control?

Ground Control is an agent-driven task orchestration system for solo developers who vibe code with AI. Instead of you managing tasks, a **Taskmaster agent** manages the flow — routing work between specialized agents and you, ensuring nothing falls through the cracks.

### The Problem with Current AI Coding

```
You: "Build me an Android app that sends messages to my TV"
AI: "Done! It's production ready!"
You: *tests it* "This doesn't work at all"
```

Ground Control fixes this by:
1. **Structured tasks** with explicit outputs and verification
2. **Agent handoffs** with context preservation
3. **Human checkpoints** where decisions actually matter
4. **Verification requirements** before anything is "done"

## Installation

### Quick Install

```bash
# Clone the repository
git clone https://github.com/mmariani/ground-control.git
cd ground-control

# Build and install to ~/.local/bin
make install

# Verify installation
gc --version
```

### Requirements

- Go 1.25+
- Claude CLI (`claude`) for AI agent execution

### Manual Installation

```bash
# Build
go build -o gc ./cmd/gc

# Copy to PATH
cp gc ~/.local/bin/

# Or install man page too
sudo cp man/gc.1 /usr/local/share/man/man1/
```

## Quick Start

```bash
# Capture an idea
gc dump "Android app to send messages to my TVs"

# See what tasks exist
gc tasks

# Run orchestration on a task (by index #)
gc orc 3

# Or by task ID
gc orc task_123456

# Check status
gc status

# Run daily standup ritual
gc standup

# Launch interactive TUI
gc tui
```

## Core Concepts

### Taskmaster Agent
The orchestrator. Knows all tasks, priorities, and your patterns. Creates tasks, routes them to appropriate agents, reviews completed work, decides what's next.

### Task Types
- **simple**: Do X (wash car, pay bill)
- **ai-planning**: Chat with planner agent to design something
- **research**: Investigate topics (can be parallel)
- **coding**: Write code with verification
- **human-input**: Need a decision from you

### The Orchestration Pipeline

```
gc orc task_123
    → Sanity Check (validate task is ready)
    → Coder (implement with Claude)
    → Reviewer (code review)
    → Tester (run tests)
    → Commit (git commit if passing)
```

Each stage can: **pass** → continue, **fail** → retry, or **escalate** → human review.

### The Flow

```
Brain Dump (idea)
    → Ingestion Agent (categorize)
    → Taskmaster (create structured task)
    → Agent executes (with verification)
    → Taskmaster reviews (decide next steps)
    → Loop until done
```

### AI Matt Delegation

Delegate decisions to AI Matt (your AI proxy) while maintaining human oversight:

```bash
# Basic delegation (5 interactions)
gc delegate --interactions 5

# Supervised mode with monitor pane
gc delegate --supervised --interactions 5

# Check status
gc delegate --status

# Cancel delegation
gc delegate --cancel
```

In supervised mode:
- Worker Claude + AI Matt run in a dedicated tmux window
- Monitor pane in your window shows communications
- Password-protected approvals (set `GC_APPROVAL_PASSWORD_HASH` in `.env`)

### Context Bundles

Every task gets a context bundle with:
- `requirements.md` — What exactly to build
- `project_context.md` — Relevant project info
- `relevant_code.md` — Code the agent needs to see
- `patterns.md` — Coding patterns to follow
- `test_hints.md` — How to verify the work

Bundles are auto-generated by the Context Engineer when missing.
